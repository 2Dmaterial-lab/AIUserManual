    数据管理与可复现科研¶

    本章给出课题组成员可直接采用的数据、代码和图表管理规范。目标不是增加负担，而是让自己三个月后还能复现结果，也让师兄师姐、同门和审稿人能看懂你的工作。

    一、目录结构¶

    建议每个独立课题或论文图建立一个项目目录：

    project_name/
├── README.md
├── raw_data/
├── processed_data/
├── scripts/
├── figures/
├── notes/
└── archive/
    目录 用途 规则
    raw_data/ 仪器、仿真或文献导出的原始数据 只读保存，不手动覆盖
    processed_data/ 清洗、筛选、归一化、拟合后的数据 由脚本生成，能重复生成
    scripts/ MATLAB、Python、VBA、LSF、LabVIEW 说明等 文件名写清任务
    figures/ 论文图、组会图、检查图 保留源文件和导出文件
    notes/ 实验记录、参数说明、AI 使用记录 使用日期命名
    archive/ 阶段性冻结版本 不放临时文件

    二、命名规范¶

    推荐格式：

    YYYY-MM-DD_sample_condition_measurement.ext

    示例：

    2026-05-19_sampleA_532nm_power_scan.csv
2026-05-19_fig2_transmission_plot.m
2026-05-19_comsol_width_sweep_results.csv

    避免：

    • 新建文件夹
    • 最终版
    • 最终版2
    • 改好了
    • data.csv

    如果必须保留多个版本，优先用日期、样品编号、参数或 Git 提交记录区分，而不是靠“最终”命名。

    三、README 最小模板¶

    每个项目目录建议放一个 README.md：

    # 项目名称

2. [脚本2]：拟合/统计/归一化
3. [脚本3]：生成图表

## 运行环境

- 操作系统：
- MATLAB/Python/Origin/COMSOL/FDTD 版本：
- 依赖包或工具箱：

```bash
[写出实际运行命令或软件操作入口]
```

    四、原始数据保护¶

    原始数据是科研结论的证据链起点。

    1. 仪器导出后先复制到 raw_data/，不要在原文件上直接修改。
    2. Excel 手动编辑容易破坏数据类型，能用脚本处理就用脚本。
    3. 重要数据至少两处备份：本机 + 移动硬盘/NAS/组内指定存储。
    4. 未发表数据、合作方数据、专利相关数据不要上传云端 AI 或公共网盘。
    5. 如果需要把数据发给 AI，优先使用小样本、脱敏数据或模拟数据。

    五、脚本可复现清单¶

    让 AI 写脚本时，要求它满足以下条件：

    项目 要求
    输入 文件路径、列名、单位写清楚
    输出 保存到固定目录，文件名可追溯
    参数 集中放在脚本开头或配置文件
    随机性 设置随机种子
    异常值 明确处理规则，不静默删除
    日志 打印关键步骤和输出路径
    版本 记录软件和依赖版本
    验证 有小样本或已知结果测试

    可复制提示词：

    请检查下面的科研数据处理脚本是否可复现。

脚本：
```[语言]
[粘贴脚本]
```

请按以下维度检查：
1. 输入文件、列名和单位是否清楚
2. 输出文件是否可追溯
3. 是否有硬编码路径或本机特有路径
4. 是否有随机性、异常值处理或单位换算风险
5. 是否需要增加日志、参数区、README 或测试数据
6. 给出修改后的脚本片段

    六、论文图可追溯清单¶

    每张论文图至少能回答：

    • 原始数据在哪里？
    • 用哪个脚本生成？
    • 图中每条曲线对应哪个样品或参数？
    • 轴标签、单位、归一化方式是否写清？
    • 拟合参数、误差棒、置信区间来自哪里？
    • 最终导出的格式和分辨率是什么？

    建议在 figures/ 下保留：

    fig2_transmission.m
fig2_transmission.pdf
fig2_transmission.png
fig2_transmission_source_data.csv

    七、AI 使用记录¶

    当 AI 参与代码、数据处理、语言润色或审稿回复时，建议在 notes/ai_usage.md 中记录：

    字段 示例
    日期 2026-05-19
    工具 ChatGPT / Claude / DeepSeek / Cursor
    用途 生成 MATLAB 绘图脚本
    输入数据 只使用脱敏小样本
    输出使用方式 人工修改后用于数据处理
    人工核查 已用测试数据和手算结果验证

    这不是为了增加形式主义，而是为了投稿、答辩、组内交接时能说清楚 AI 到底做了什么。

    八、交接包¶

    毕业、换方向或把项目交给同门前，建议整理一个最小交接包：

    1. README.md
    2. 原始数据说明，不一定复制全部大文件，但要写清存放位置
    3. 关键处理脚本
    4. 最终图表源数据
    5. 仪器/仿真参数表
    6. 未解决问题和注意事项
    7. 需要权限的软件、账号或许可证说明

    九、与 AI 协作的底线¶

    • 不把未公开成果、完整原始数据、专利细节直接输入云端 AI。
    • 不让 AI 编造数据、文献、实验条件或审稿回复事实。
    • 不把“AI 说可以”作为物理模型、统计方法或仪器安全设置的依据。
    • 任何用于论文结论的代码和图表都必须能复现、能解释、能追溯。
