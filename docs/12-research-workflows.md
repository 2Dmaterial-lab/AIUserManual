# 科研任务工作流

本章按研究生常见任务组织内容：先明确目标，再选择工具和 AI 协作方式。遇到具体软件操作时，再跳转到对应工具章节。

!!! warning "基本原则"

    AI 可以帮助你检索线索、生成脚本、整理表格和检查逻辑，但科研结论必须回到原始数据、论文原文、仪器记录和仿真模型中验证。

## 一、从文献调研到综述表 {: #literature-evidence }

适用场景：开题、组会汇报、论文引言、寻找可复现实验路线。

### 推荐流程

1. **拆解研究问题**：让 AI 把题目拆成材料体系、物理机制、制备方法、表征手段和应用场景。
2. **生成检索式**：让 AI 给出中英文关键词、同义词、排除词和数据库检索式。
3. **正式检索**：在 Web of Science、Scopus、Google Scholar、CNKI、arXiv、期刊官网等来源检索。
4. **建立证据表**：记录题目、作者、年份、DOI、样品、方法、关键结果、局限性、与你课题的关系。
5. **回到原文核查**：对关键参数、图表数值、实验条件和结论边界逐条核对。

### 可复制提示词

````
我正在做文献调研，研究问题是：[研究问题]。

请帮我拆解检索策略：
1. 给出 5 组英文关键词组合和 5 组中文关键词组合
2. 给出可能的同义词、缩写和排除词
3. 按“材料/结构/机理/表征/应用”分类
4. 建议我优先查哪些数据库
5. 输出一个文献证据表模板

注意：不要编造具体论文。只生成检索策略和表格字段。
````

### 证据表字段

| 字段 | 说明 |
|------|------|
| 论文信息 | 标题、作者、年份、期刊、DOI |
| 研究对象 | 材料、结构、器件、样品编号 |
| 方法 | 制备、测试、仿真、统计方法 |
| 关键结果 | 数值、单位、图号、实验条件 |
| 局限性 | 样本量、适用范围、未解决问题 |
| 可复用内容 | 参数、流程、代码、图表样式 |
| 核查状态 | 已读全文 / 只读摘要 / 待核对 |

## 二、从原始数据到论文图 {: #data-to-figure }

适用场景：仪器导出 CSV/TXT/Excel 后，需要清洗、拟合、绘图并生成可投稿图。

### 推荐目录

```text
project_name/
├── raw_data/          # 原始数据，只读保存
├── processed_data/    # 清洗、归一化、拟合后的数据
├── scripts/           # MATLAB/Python/Origin 脚本
├── figures/           # 输出图
└── README.md          # 数据来源、处理步骤、软件版本
```

### 推荐流程

1. 复制原始数据到 `raw_data/`，不要直接覆盖仪器导出的文件。
2. 用 AI 生成数据读取和清洗脚本，明确列名、单位、异常值处理规则。
3. 先用小样本运行，检查单位、行列方向、缺失值和坐标轴。
4. 生成 `processed_data/`，保留中间结果，避免只保留最终图片。
5. 用脚本生成论文图，记录字体、尺寸、线宽、颜色、导出格式。
6. 用人工和 AI 双重检查：坐标轴单位、图例、误差棒、拟合参数和图注是否一致。

### 可复制提示词

````
请帮我把下面的数据处理需求整理成可复现脚本。

研究目标：[要回答的问题]
输入文件：[文件名/路径/格式]
列含义与单位：
- [列1]：[含义，单位]
- [列2]：[含义，单位]

处理步骤：
1. [清洗/筛选/归一化/单位换算]
2. [拟合/统计/计算]
3. [绘图]

输出要求：
- 保存清洗后的数据到 processed_data/
- 保存图到 figures/
- 图像尺寸：[宽 x 高]
- 字体、线宽、颜色：[要求]

请同时输出：
1. 脚本
2. 输入数据假设
3. 可能出错的地方
4. 结果验收清单
````

## 三、从仿真参数扫描到结论 {: #simulation-sweep }

适用场景：COMSOL、FDTD、MATLAB 等工具做参数扫描、优化和结果汇总。

### 推荐流程

1. 写清楚科学问题：扫描参数是为了验证趋势、寻找最优值，还是排除某种解释。
2. 固定基准模型：材料、几何、边界、网格、源、监视器和求解器参数先保存一份。
3. 设计参数表：参数名、单位、范围、步长、默认值、物理依据。
4. 批量运行前先做 2-3 个测试点，确认结果方向和导出字段正确。
5. 扫描结果保存为结构化表格，不只保存截图。
6. 做网格无关性、边界条件敏感性或重复运行检查。
7. 把最终图、参数表和模型版本一起归档。

### 参数表模板

| 参数 | 单位 | 默认值 | 扫描范围 | 步长 | 物理依据 | 备注 |
|------|------|--------|----------|------|----------|------|
| `width` | nm | 500 | 400-700 | 20 | 工艺线宽范围 | 示例 |
| `height` | nm | 220 | 180-260 | 10 | 薄膜厚度范围 | 示例 |

### 可复制提示词

````
我准备做 [COMSOL/FDTD/MATLAB] 参数扫描，请帮我检查方案是否合理。

研究目标：[说明要验证的科学问题]
基准模型：[材料、几何、边界、源、监视器/探针]
计划扫描参数：
[贴出参数表]

请帮我：
1. 检查参数范围和步长是否可能遗漏关键区域
2. 指出需要固定不变的参数
3. 建议最小测试点和网格/边界敏感性检查
4. 设计结果导出表格字段
5. 给出失败任务的重跑策略
````

相关章节：[COMSOL](05-comsol.md)、[FDTD](06-fdtd.md)、[MATLAB](03-matlab.md)。

## 四、从仪器采集到自动化记录 {: #instrument-automation }

适用场景：LabVIEW、Python、MATLAB 控制仪器，自动采集数据并保存。

### 推荐流程

1. 先手动确认仪器地址、通信协议、量程、采样率、积分时间和安全限制。
2. 让 AI 生成状态机或伪代码，不要一开始就生成完整控制程序。
3. 对每条 SCPI 命令或驱动 API 回到仪器手册核对。
4. 每次采集保存元数据：仪器型号、序列号、软件版本、参数、时间、操作者。
5. 程序必须包含超时、重试、停止按钮、异常日志和安全默认状态。
6. 首次自动运行时使用假负载或低风险参数。

### 最小元数据

| 字段 | 示例 |
|------|------|
| 日期时间 | `2026-05-19 20:30:00` |
| 操作者 | 姓名或学号 |
| 仪器 | 型号、序列号、连接方式 |
| 软件 | LabVIEW/MATLAB/Python 版本 |
| 参数 | 电压、电流、波长、积分时间、采样率 |
| 文件 | 原始数据路径、脚本路径 |
| 备注 | 异常、校准状态、环境条件 |

相关章节：[LabVIEW](09-labview.md)、[MATLAB](03-matlab.md)、[Excel](04-excel.md)。

## 五、投稿前检查 {: #submission-check }

适用场景：论文、会议摘要、补充材料、审稿回复提交前。

### 检查清单

1. **数据一致性**：正文、图、表、补充材料中的数值和单位一致。
2. **图表可追溯**：每张图能追溯到原始数据、处理脚本和导出命令。
3. **引用真实**：DOI、页码、作者、年份逐条从数据库或期刊官网核对。
4. **方法可复现**：关键参数、仪器型号、软件版本和统计方法写清楚。
5. **AI 使用记录**：如果期刊或学校要求声明，准备工具、用途、日期和人工核查说明。
6. **敏感信息**：未公开数据、专利细节、合作方信息和个人信息已按要求处理。

### 可复制提示词

````
请作为投稿前检查助手，帮我审查下面这段论文材料。

材料类型：[摘要/引言/方法/结果/图注/审稿回复]
目标期刊或会议：[名称，如未知可写“待定”]
文本：
[粘贴内容]

请检查：
1. 是否有逻辑跳跃或过度结论
2. 单位、变量名、图表编号是否一致
3. 哪些引用或事实必须回原文核对
4. 哪些地方需要补充实验条件、统计方法或不确定性
5. 是否涉及 AI 使用声明或数据合规风险

不要替我编造数据、引用或实验结果。
````
